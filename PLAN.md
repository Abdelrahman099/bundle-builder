# Wyze Bundle Builder — Build Plan

A multi-step bundle builder with a live review panel, rebuilt from Figma to a
pixel-perfect, data-driven React + TypeScript + Tailwind v4 prototype.

Figma: https://www.figma.com/design/JYf61etQVqeseX7oY5alGz/Frontend-Test-Figma?node-id=68-8088
Reference frames: Frame 1735 = desktop (1440px), Frame 1736 = large desktop, iPhone 13 & 14 = mobile (local exports, not committed).

---

## 1. Locked decisions

| Area | Decision | Rationale |
|---|---|---|
| Language | **TypeScript** (strict) | The app is data → derived state; types on the variant-keyed cart prevent the exact bugs this task tests. |
| State | **Zustand** — single store, selector subscriptions | Many components read/write one cart. Selectors let each stepper subscribe to only its variant's qty, so the total recalcs without re-rendering every card. |
| Styling | **Tailwind v4** with a `@theme` token block | CSS-first tokens (brand colors, radii, fonts) defined once, composed via utilities; easy to pixel-tune. |
| Data | **`src/data/products.json`** + typed loader | Requirement: data-driven, no hardcoded per-product markup. Same JSON drives cards and review. |
| Pricing | **Compute from unit price** (`unit x qty`) | JSON stores unit price + compareAt; all lines/totals/savings computed live. Honest math; may differ a few cents from the mock's hand-typed $187.89 — documented in README. |
| Persistence | **localStorage**: continuous autosave + explicit "Save my system for later" | Autosave guarantees reload restores; Save link writes the snapshot and confirms. |
| Icons/images | `public/icons.svg` sprite or `lucide-react`; product photos exported from Figma | Step icons + product photos referenced from JSON. |

## 3. Folder structure

```
src/
  main.tsx
  app/App.tsx                 # shell: <Builder/> + <ReviewPanel/>
  data/products.json          # single source of truth
  lib/
    types.ts                  # Product, Variant, Category, CartState
    pricing.ts                # lineTotal, subtotal, savings, financing (pure)
    format.ts                 # currency + "FREE"
  store/
    useBuilderStore.ts        # quantities map, activeVariant, openStep, actions
    selectors.ts              # selectedCountByStep, reviewGroups, totals
  hooks/usePersistence.ts     # hydrate + autosave to localStorage
  components/
    ui/       Button Badge Price QuantityStepper VariantChip Chevron Divider StepBadge
    builder/  Builder AccordionStep StepHeader ProductGrid ProductCard VariantSelector
    review/   ReviewPanel ReviewGroup ReviewLine SummaryRows TotalBlock
  styles/index.css            # @import tailwind + @theme tokens
```

Principle: `ui/` = generic, stateless. `builder/` + `review/` = feature components
reading the store. All product markup generated by mapping data.

## 4. Data model (variant-keyed quantities)

```ts
type Category = 'cameras' | 'sensors' | 'accessories' | 'plan';

interface Variant { id: string; label: string; swatch: string; price: number; compareAt?: number }
interface Product {
  id: string; step: 1|2|3|4; category: Category;
  title: string; description: string; image: string;
  badge?: string; learnMore?: boolean;
  variants: Variant[];      // no-variant products get one { id:'default' }
  addable: boolean;         // review-only seed items (Hub/plan) have no add control
}

// single source of truth
type VariantKey = `${string}:${string}`;   // 'wyze-cam-v4:white'
quantities: Record<VariantKey, number>;     // Red & Blue counted separately
activeVariant: Record<string, string>;      // productId -> current chip
openStep: 1|2|3|4;
```

Card stepper reads `quantities[`${productId}:${activeVariant[productId]}`]`.
Review lists every key with qty > 0. Both read the same map -> always in sync.

Derived (memoized): `selectedCount(step)` = distinct products with qty>0;
`reviewGroups` grouped by category; `totals` / `savings` / `financing`.

## 6. Responsive strategy

- >=1280 (Frame 1735): `grid: minmax(0,2fr) minmax(320px,1fr)`; sticky review;
  cards `repeat(auto-fit, minmax(300px,1fr))`.
- 768–1279 (Frame 1736): builder full-width, cards wrap to a row; review below,
  full-width, internal 2-col.
- <768 (mobile): single column; "Let's get started!" header; all steps collapsed;
  review stacked; guarantee inline.

One `auto-fit minmax` card grid yields both the 2-col and single-row layouts.

## 8. Build sequence

1. Tokens + `products.json` + types + pricing utils.
2. `ui/` primitives (Button, Stepper, Badge, Price, VariantChip).
3. Store + selectors + persistence hook.
4. ProductCard -> ProductGrid -> AccordionStep -> Builder (match Frame 1735).
5. ReviewPanel + groups + totals (match design).
6. Wire variant/stepper sync + counters + live total.
7. Responsive passes + Save persistence + Checkout confirmation.
8. Polish, a11y (keyboard accordion, aria steppers), README with decisions/tradeoffs.
